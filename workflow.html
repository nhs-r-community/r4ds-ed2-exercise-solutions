There are a lot of corrections needed in this code. It includes:

  • todyverse should be tidyverse)
  • dTA instead of data)
  • maping instead of mapping
  • missing , between x = displ and y = hwy
  • a " before "lm
Quotations around the lm in method = (note)

"lm" will work but RStudio will flash up an error in the margin saying unexpected token if you click on the red circle with white cross.

  • geom_smooth() also requires its own mapping information as the aes() isn’t available to that line. If this were global it would be:
library(tidyverse)

ggplot(data = mpg, mapping = aes(x = displ, y = hwy)) + 
  geom_point() +
  geom_smooth(method = lm)

  1. Let’s revisit an exercise from the https://r4ds.hadley.nz/data-visualize#sec-ggsave. Run the following lines of code.

my_bar_plot <- ggplot(mpg, aes(x = class)) +
 geom_bar()
my_scatter_plot <- ggplot(mpg, aes(x = cty, y = hwy)) +
 geom_point()
ggsave(filename = "mpg-plot.png", plot = my_bar_plot)

Which of the two plots is saved as mpg-plot.png?

The bar plot from the object my_bar_plot is saved.

Why?

This has been explicitly detailed in the ggsave() function. Both plots are also saved as objects (https://r4ds.hadley.nz/workflow-basics#sec-whats-in-a-name) so they can be referred to in the function.
